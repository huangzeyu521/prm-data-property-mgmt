# 数据产权管理模块 优化方案
> 依据:《附录F 中国南方电网有限责任公司数据确权授权业务指导书》、《数据确权授权工作指引》。
> 方法:逐条对照"治理要求 → 现状实现 → 差距 → 优化动作",按功能模块给出。日期 2026-06-02。

## 0. 总纲:六大原则合规自检

| 原则(附录F 3.2) | 现状 | 差距 | 优化方向 |
|---|---|---|---|
| 三权分置 | 档案/卡片有 rightType 单值 | 未按"持有/使用/经营"三权分别确权、分别发证;经营权管制/非管制规则缺失 | 一资产可并存三权档案;经营权按管制类/非管制类走不同授权与备案 |
| 先确后授 | 已实现(授权校验权益卡片,冻结熔断) | 校验仅"卡片可用",未校验"经营权授权范围⊆对外开放目录" | 授权侧增加"对外开放目录"与"授权≤确权边界"双校验 |
| 权益归集 | 卡片/档案归集 | 网省地多级组织协同、公司统筹归集未建模 | 按网/省/地三级组织 + 公司统筹的归集口径 |
| 业务导向 | 分类分级字段已同步 | 未按分类分级驱动授权审批深度 | 审批深度随数据敏感级动态(已在需规,未落地) |
| 权益精细化 | 无第三方权益建模 | 缺表2(涉及第三方权益)、第三方许可凭证 | 增第三方权益识别/凭证/保密承诺函 |
| 权益可追溯 | 变更记录自动留痕 | 上链存证仍为 TODO;认定意见/审核意见未结构化全留痕 | 各节点留痕 + 区块链存证落地 |

## 1. F-02 数据确权管理(对照附录F 4.1 八节点流程)

**治理流程(附录F 4.1.3)**:10 组织发起(公司数字化部主管)→20 组织收集/编制资料(网省地数字化部,表1/表2+证明材料)→30 业务部门配合提供→40 归集/审查完整性→50 合规管控小组审核(生成表3 确权信息汇总表、表4 数据权益内部管理汇总表 + 权益认定意见)→60 主管审核结果清单及认定意见→70 经理/高级经理审批→80 制作数据权益卡片、确权记录归集。

**现状**:`ConfirmApply` 状态机仅 草稿→合规审核中→主管复核中→已完成(2 次 approve)+ 终审自动生成权益卡片;有 指引/材料+校验/证书+模板。

**差距与优化**:
1. **节点对齐**:把状态机扩展为与治理一致的节点:`草稿→已提交(10/20)→资料归集审查(40)→合规审核(50)→主管复核(60)→经理终审(70)→已制卡(80)`,每节点绑定治理角色(主管/网省地团队/业务部门/合规管控小组/经理),记录处理人/时间/意见。
2. **表1/表2 双清单**:确权申请区分《表1 确权信息清单(系统级)》与《表2 确权信息清单(涉及第三方权益)》;新增**第三方权益识别**字段(是否涉及第三方、第三方机构、权益转移约定、许可凭证)。
3. **表3/表4 自动汇总**:在"合规审核(50)"节点由合规小组自动生成《表3 确权信息汇总表》《表4 数据权益内部管理汇总表》与结构化**权益认定意见**(非纯文本)。
4. **三权分置出证**:一资产可分别确权 持有权/加工使用权/产品经营权,各自生成权益卡片与证书;经营权确权区分管制类/非管制类。
5. **季度重确权(附录F 3.3.2)**:新增"权益变更/到期"触发 + **季度定时重确权**任务(数据新增/来源变更/管理要求变更/到期),生成变更确权工单。
6. **数据权益风险表(附录F 3.3.3 + 附录F表)**:合规小组对持有/使用/经营权风险评估(如"用户笼统授权""第三方未约定权益转移"),填报《数据权益风险表》并纳入审核。
7. **AI 辅助(需规+AI集成)**:材料 OCR 要素抽取、权属冲突识别(多头确权/范围重叠)、RAG 决策建议接入大瓦特(现为桩)。

## 2. F-03 数据授权管理(对照附录F 4.2 批量 / 4.3 专项)

**治理流程**:
- **专项一事一议(4.3)**:发起申请(网省地业务部门,表5)→申请审查→合规性审核(合规管控小组)→授权申请审核(公司业务部门经理/高级经理)→授权申请审批(数字化部主管→经理/高级经理→副总/总经理)→授权方签协议→被授权方签协议→执行授权。
- **批量(4.2)**:发起征集(公司数字化部主管)→组织收集需求(表5)→配合提交→归集/审查→资料审核(合规小组)→《表6 数据批量授权清单》草案→申报稿审核审批→**领导小组办公室决策批准**→授权方签协议→被授权方签协议→执行授权。
- 授权方式两类、分级授权(跨区域/跨业务域/管制类由公司授权;管制类分子公司向省级授权并报备;非管制类备案,附录G)。

**现状**:`AuthApply` 单级 草稿→审核中→已生效(1 次 approve 即发证);先确后授校验已实现;`AuthAgreement` 单方 签章→审核→存档;`AuthCompliance` 红黄绿;catalog 覆盖 指引/场景/表单模板/协议模板库。

**差距与优化**:
1. **多级审批链**:按治理重构审批:
   - 专项:`合规性审核(合规小组)→业务部门经理审核→数字化部主管审核→经理/高级经理审核→副总/总经理审批`(5 级),每级可驳回/退回。
   - 批量:`合规小组审核→数字化部认定→领导小组办公室批准`,新增**领导小组办公室**审批节点。
2. **表5/表6**:专项用《表5 数据授权申请单》;批量需将各单位表5**归集形成《表6 数据批量授权清单》**(草案→申报稿→批准三态)。当前缺表6 聚合与状态机。
3. **协议双签前置(附录D)**:把"授权方签 + 被授权方签"双签作为**执行授权/签发证书的前置**;`AuthAgreement` 增加甲方/乙方分别签署、签署时间、签章有效性(CA/CV)。当前为单方,需补双签。
4. **备案(附录G)**:分子公司对外数据经营权授权,新增**对外经营授权备案**对象(附录G 备案表)与"协议+清单在数字化部备案"步骤。
5. **保密承诺函(附录E)**:涉敏感/第三方数据,授权申请强制附《保密承诺函》。
6. **授权原则校验**:经营权授权范围**仅限对外开放目录**——授权提交时增加"资产∈对外开放目录"校验。
7. **动态跟踪(附录F 3.4.5)**:授权**期限管理**——到期前预警、**续签/终止**流程;履约**违规/安全事件→暂停授权(联动底层权限回收)+追责工单**,风险解除后恢复/终止。当前仅证书撤销,需补续签与暂停-追责闭环。
8. **合规监管(附录F 3.4.6)**:取得经营权对外提供数据产品/服务需**备案 + 自行安全合规评审**记录。

## 3. F-01 产权信息管理 / 权益动态监测

**治理依据**:权益可追溯(3.2.6)、权益变更(3.3.2)、风险处置(3.3.3)、动态跟踪(3.4.5)。

**现状**:产权档案/树/概览/变更记录/统计;监测规则/预警处置闭环/到期合规巡检。

**差距与优化**:
1. **权益链条完整留痕**:变更记录已自动生成,但需覆盖治理要求的全要素(确权证明、判定意见、审核意见、权益转移、授权需求/审核/协议),并**区块链上链存证**(落地现 TODO)。
2. **季度重确权联动**:监测侧识别"数据新增/来源变更/到期"→自动派生 F-02 重确权工单(对应 3.3.2)。
3. **风险预警对齐风险表**:监测预警与《数据权益风险表》打通,风险项→合规小组处置→留痕。
4. **权益动态监测覆盖授权履约**:授权到期、越权调用、违规使用纳入监测,联动 F-03 暂停授权+追责(对应 3.4.5)。
5. **统计口径对齐治理 KPI**:全网确权覆盖率、各单位高价值数据授权量、按时响应率/整改闭环率作为效能考核指标(需规已列)。
