# 确权申请(一站式) 前端流程测试手册

> 已用本目录材料通过 API 全链路预验证（2026-06-11）：自动填充→草稿→表级清单→6 份材料上传→校验全过→推送审核成功。照此手册在界面操作必可跑通。

## 测试 ID

| 用途 | 关联资产ID | 说明 |
|---|---|---|
| **正向流程（推荐）** | `AST-001` | 台账种子资产"客户用电信息表"（广东电网/营销2.0），元数据质量分 **92**≥80 |
| 其他可用 | `AST-002` ~ `AST-008` | 台账其余种子资产，质量分均 92 |
| **负面用例：低质量驳回** | `AST-LOWQ-1` | 任何含 `LOWQ` 的 ID 质量分 60 <80，提交时**自动驳回** |

## 步骤 1：填写申请（逐字段值）

1. 关联资产ID 输入 **AST-001** → 点"元数据自动填充"（或失焦自动同步）
   - 预期：质量评分 tag 显示 **92**（绿色）；资产名称/申报主体/责任部门自动带出
2. 资产名称：改为 **客户用电信息表**
3. 权属类型（多选）：**数据资源持有权 + 数据加工使用权**
4. 申报权属主体：广东电网有限责任公司（自动带出可不改）
5. 责任部门：数字化部；系统负责人：张工；联系方式：020-88886666
6. 登记类型：**初始确权**；申请模式：**常规**；管制属性：**管制业务**
7. 来源权益识别：勾 **A 自行生产**
8. 信息关联识别：勾 **G 行政监管 + H 个人/家庭隐私**
   - 预期：出现"表2 涉及第三方权益"区块（**必填，否则提交审核被门禁拦截**）
9. 表2 字段：
   - 来源主体名称：**用电客户**
   - 来源权益限制摘要：涉个人信息字段对外提供须脱敏并经客户授权
   - 信息识别关联主体：国家能源局南方监管局;用电客户
   - 权益风险说明：未经授权对外提供个人信息存在合规风险
10. 用途说明：营销域购售电数据确权（测试）
11. 表级数据清单 批量粘贴（粘贴后点"解析为表级清单"，预期解析 1 行）：
    ```
    MKT_DB01,MKT,C_CONS_ELEC_INFO,客户用电信息表,敏感信息,A 自行生产数据,广东电网有限责任公司
    ```
12. 点 **下一步:上传材料**
    - 预期：提示"申请已暂存(MDAU-00-…)"——编号为 MDAU 工单格式；提示"已保存 1 张表级清单(M02)"

## 步骤 2：上传材料（应交清单 6 项 ↔ 本目录文件一一对应）

| 应交材料 | 上传本目录文件 |
|---|---|
| 《表1 数据确权信息清单(系统级)》 | `01-表1-数据确权信息清单(系统级)-AST-001.docx` |
| 数据确权证明材料(权属/来源凭证) | `02-数据确权证明材料-权属来源凭证-AST-001.docx` |
| 《表2 数据确权信息清单(涉及第三方权益)》 | `03-表2-…-AST-001.docx` |
| A：数据来源设备/系统建设投入情况说明 | `04-A-数据来源系统建设投入情况说明-AST-001.docx` |
| G：行政监管要求补充说明 | `05-G-行政监管要求补充说明-AST-001.docx` |
| H：个人/家庭隐私授权说明 | `06-H-个人家庭隐私授权说明-AST-001.docx` |

每行点"上传原件"选对应文件 → 状态变"已上传"（绿色）。6 项全部上传后点 **下一步:材料校验**。
（可选：点"调用智能确权辅助工具"按钮，新标签打开 AI 工具并自动带本申请 ID。）

## 步骤 3：材料校验

1. 点 **规则校验全部材料**
   - 预期：绿色提示 **"应交 6 项,已交 6,通过 6,缺失 0,不合规 0 — 完整且合规,可推送审核"**
2. （可选）点 **AI 智能校验**：返回 AI 结论；若提示需先解析材料，点"进入智能确权辅助工具"上传解析后再试
3. 查看 **权益归集判定** 面板
   - 预期：命中 **规则 3.1**（管制+涉 G/H 第三方）；网公司持有/使用=有/有，经营权=**依权益判定**；共享判定原因以"根据公司确权授权工作指引…"开头
4. 查看审批链四节点：合规管控小组审核 → 主管复核 → 经理/高级经理终审 → 制卡归集
5. 点 **提交审核**
   - 预期：进入步骤 4"确权申请已提交"，申请状态变 **合规审核中**

## 步骤 4 之后：审批闭环（可选延伸）

- 左侧菜单 **申请查询**：可见本申请（MDAU 编号），进度=合规审核中；本页还可点"导出《数据确权信息汇总表》/《权益内部管理汇总表》"验证官方汇总表（含刚录入的表级清单行）
- 左侧菜单 **审核申请提交**：以审核员视角逐级通过 合规→主管→经理，最终生成权益卡片（权益卡片生成页可见）

## 负面用例

| 用例 | 操作 | 预期 |
|---|---|---|
| 低质量元数据驳回 | 资产ID 用 `AST-LOWQ-1` 走完流程提交 | 状态=已驳回，原因"元数据质量评分 60 低于 80,自动驳回" |
| 表2 门禁 | 勾了 G/H 但表2"来源主体名称"留空 | 步骤1 下一步被拦："须填表2来源主体名称"；绕过后提交审核报"须填写表2第三方权益信息" |
| 格式校验 | 上传 .txt/.exe 文件 | 上传被拒（仅 PDF/Word/JPG/PNG） |
| 缺材料提交 | 只传 5 份就校验+提交 | 校验提示缺失 1 项，提交被门禁拦截 |
