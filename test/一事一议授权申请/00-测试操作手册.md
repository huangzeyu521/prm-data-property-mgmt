## 步骤 2：合规校验

点 **执行合规校验**，预期 **结果=通过、风险等级=绿**，7 个校验项全部通过：
材料完整性(已上传 5 份) / 先确后授(已引用权益卡片) / 授权权益类型 / 授权范围 / 第三方许可凭证(不涉及) / 敏感数据保护(已含保密承诺+信息授权协议) / 跨区域授权(非跨域)。

## 步骤 3-4：提交审核 → 完成

提交后申请状态 = **合规审核中**，进入多级审批（合规小组→业务部门→数字化部→经理/副总）。
后续闭环：**授权审核提交**页逐级审批 → **协议签章上传**页上传 05 号文件的双签扫描件 → 自动签发授权证书（**授权权益管理**页可见）。

## 负面用例

| 用例 | 操作 | 预期 |
|---|---|---|
| 冻结卡片熔断 | 权益卡片ID 改 `FROZEN-EC-001` 走到合规校验 | 校验**不通过**（权益卡片不可用/已冻结） |
| 范围越界 | 卡片 `EC-NARROW-001` + 授权范围"全字段" | 不通过（授权范围超出确权边界"约定字段"） |
| 时效越界 | 卡片 `EC-SHORT-001` + 有效期 2028 年 | 不通过（超出确权有效期） |
| 第三方缺凭证 | 第三方来源方式填"数据采购"但许可凭证留空 | 表单必填拦截/合规不通过 |
| 缺保密承诺 | 涉个人隐私但关闭保密承诺函 | 合规校验"敏感数据保护"不符 |

## 测试中发现并已修复的 bug

向导"有效期"日期选择器原 value-format 为 ISO `T` 格式，后端 LocalDateTime 仅接受 `yyyy-MM-dd HH:mm:ss`——界面选日期必报"请求体格式错误"。已修复（AuthWizard.vue / AuthCertList.vue），现可正常选择。
