# 智能确权辅助工具 · 自动测试材料

由 `TestMaterialGenerator` 自动生成(复用 POI/PDFBox/ImageIO),覆盖可研 1.1.1.1~3.3 九大组的测试场景。
每个文件的场景与**预期解析结果**见 `INDEX.json`。

> 容器无中文字体:Word/Excel 为真中文(走文本抽取);PDF 文本层为 ASCII;扫描版 PDF 与 PNG 为图片页(走 OCR,汉字像素可能为缺字框,实际内容以 INDEX.json 为准)。

## 目录
| 目录 | 用途 |
|---|---|
| 01_格式覆盖 | Excel/Word/PDF文本层/扫描PDF/图片 五格式 |
| 02_数据清洗 | 脏数据(噪声/全半角/空值/非标枚举/重复/布尔)+ 自定义字段映射 |
| 03_要素抽取 | 五类主体+五类约束+来源方式+数据特征(含个人信息/商密分级) |
| 04_批量解析 | 同批多文件 |
| 05_冲突场景 | 历史 vs 当前 对立权属主张 |
| 06_重复检测 | 同内容副本(内容指纹查重) |
| 07_异常负面 | 无章/空内容(负面用例) |

## 使用
- 人工:前端 智能确权辅助工具 → 上传材料,逐场景验证。
- 自动:`TestMaterialConsumeTest`(-Dverify.base=)把每个文件喂回 uploadBinary+parse 断言。
